## レビュー観点（優先度順）

1. 正しさ: ロジックバグ、エッジケース未処理、off-by-one
2. 安全性: SQLインジェクション、unwrap()、panicの可能性、秘密情報ハードコード
3. エラー処理: エラーが適切に伝播しているか、握りつぶしていないか
4. テスト: 変更に対応するテストがあるか（テストトロフィー基準）
5. パフォーマンス: N+1クエリ、不要なclone()、不要な再レンダリング
6. 規約準拠: CLAUDE.md / rules の規約に違反していないか

## 出力フォーマット

ファイルごとにまとめる:

```
## backend/src/routes/items.rs

[critical] L42: unwrap() がプロダクションコードにある。
  ? 演算子に置き換え、AppError を返す。

[warning] L67-72: N+1クエリの可能性。
  ループ内でDB問い合わせしている。JOINまたはバッチクエリに変更する。

[suggestion] L15: clone() が不要。参照で渡せる。
```

深刻度:
- [critical] — マージ前に必ず修正
- [warning] — 修正を強く推奨
- [suggestion] — 改善提案

## 手順

1. `git diff` で変更差分を取得する
2. 変更されたファイルを全文読む（差分だけでは文脈が足りない）
3. 上記観点でレビューする
4. 問題がなければ「LGTM」と明示する
